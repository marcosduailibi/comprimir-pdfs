Processamento local

Privacidade no ArqKit

Entenda como seus arquivos são processados no navegador e quais dados ficam no seu dispositivo.

Resumo rápido

O ArqKit foi projetado para funcionar localmente no navegador. Nas ferramentas prontas, seus arquivos são selecionados no seu dispositivo e processados com JavaScript e APIs do próprio navegador. O ArqKit não envia seus PDFs, imagens, vídeos, áudios ou documentos para servidores próprios.

O que significa processamento local

Processamento local significa que o arquivo é lido no navegador depois que você o seleciona. A ferramenta usa APIs como File API, Canvas, Web Workers e bibliotecas JavaScript carregadas no navegador para gerar o resultado final. O arquivo final é disponibilizado para download no próprio dispositivo.

Seus arquivos são enviados?

Não nas ferramentas prontas do ArqKit. Os arquivos não são enviados para backend do ArqKit; eles são processados na aba do navegador. Se alguma ferramenta futura exigir servidor, ela deverá ser claramente marcada antes do uso, com aviso específico. Nenhuma ferramenta deve enviar arquivos de forma silenciosa.

Bibliotecas por CDN

Algumas bibliotecas podem ser carregadas por CDN para executar funções no navegador, como manipular PDFs, imagens, vídeos ou QR Codes. Carregar uma biblioteca por CDN não significa enviar seus arquivos para essa CDN. A biblioteca é baixada pelo navegador e executada localmente.

O que pode ficar salvo

O ArqKit pode salvar apenas preferências simples, como tema claro/escuro, favoritos, ferramentas usadas por último, frequência de uso e preferências visuais. Esses dados podem ser armazenados em localStorage.

O que não deve ser salvo

  • PDFs enviados pelo usuário;
  • imagens de documentos;
  • vídeos e áudios;
  • nomes de arquivos e metadados;
  • thumbnails;
  • texto extraído por OCR;
  • senhas;
  • conteúdo de formulários ou documentos.

Arquivos temporários

Algumas ferramentas podem usar memória do navegador, Object URLs, IndexedDB ou OPFS para processar arquivos grandes. Esses dados devem ser temporários e limpos ao concluir, cancelar, fechar a ferramenta ou clicar em "Limpar tudo".

Câmera para PDF

A ferramenta Câmera para PDF pede permissão antes de acessar a câmera. Ela funciona principalmente em dispositivos móveis, mas continua sendo uma ferramenta de navegador. As fotos capturadas são processadas localmente para gerar PDF e não são enviadas para servidor. A câmera deve ser desligada ao sair da ferramenta.

Ferramentas em beta ou em breve

Ferramentas marcadas como Beta podem ter limitações. Ferramentas marcadas como Em breve ainda não aceitam arquivos nem simulam processamento. O ArqKit deve ser transparente sobre o que já funciona e o que ainda está em desenvolvimento.

Vídeo, imagem e áudio

As ferramentas de imagem, vídeo e áudio podem usar Canvas, WebCodecs, FFmpeg/WASM ou outras bibliotecas carregadas no navegador. Ferramentas pesadas devem ser marcadas como Beta ou Pesada e podem demorar para carregar em dispositivos mais simples.

Transparência do código

Quando disponível, o código pode ser consultado no GitHub para verificar como as ferramentas funcionam. O objetivo é manter o processamento transparente, local e auditável.
